Clifft GPU Backends

CUDA vs HIP (AMD ROCm)

How the same bytecode interpreter runs on NVIDIA and AMD GPUs
for billion-shot quantum error correction sampling

Audience: GPU programmers familiar with CUDA / HIP programming models

1. Why GPU for Quantum Circuit Simulation?

  • The SVM bytecode interpreter is simple enough to run per-thread on GPU
  • Use case: need billions of shots for rare-event estimation (logical error rate ~10-9)

2b. Host-Device Boundary

The CPU-side Clifft library is used only for compilation. The GPU kernel reimplements the SVM bytecode interpreter in CUDA/HIP.

For typical QEC circuits, k = 1–3, so the amplitude vector is just 2–8 complex floats — fits in registers. The PRNG state (32 bytes) is the dominant per-thread allocation.

3b. Three Kernel Modes

The sampler selects kernel mode based on peak_rank (the maximum number of active qubits at any point):

Modepeak_rankAmplitude StorageParallelism Model
sample_kernel ≤ 4 Registers / local memory 1 thread = 1 shot
sample_kernel_coop 5 – 10 Shared memory (up to 1024 amplitudes) 1 block = 1 shot (threads cooperate)
sample_kernel_global_coop 11 – 19 Global memory (up to 524K amplitudes) 1 block = 1 shot, persistent kernel

Key Differences from CPU

  • No ISA-specific SIMD kernels — GPU has its own SIMT model
  • 1-qubit gates: operate on amplitude pairs with stride 2i
  • 2-qubit gates: operate on 4-tuples with strides 2i, 2j
  • When k is small (1–3), these are just a few multiplies per thread

9. Build System Integration

CUDA Build

option(CLIFFT_CUDA_ENABLE_CUDA ... OFF)

if(CLIFFT_CUDA_ENABLE_CUDA)
  enable_language(CUDA)
  set(CMAKE_CUDA_ARCHITECTURES native)
  find_package(CUDAToolkit REQUIRED)
  target_link_libraries(
    clifft_cuda PRIVATE CUDA::cudart)
endif()

# Compiler: nvcc
# Source:   cuda_sampler.cu

HIP Build

option(CLIFFT_AMD_ENABLE_HIP ... OFF)

if(CLIFFT_AMD_ENABLE_HIP)
  enable_language(HIP)
  set(CMAKE_HIP_ARCHITECTURES gfx942)
  find_package(hip REQUIRED)
  target_link_libraries(
    clifft_hip PRIVATE hip::device)
  target_compile_options(
    clifft_hip PRIVATE -ffp-contract=off)
endif()

# Compiler: hipcc
# Source:   hip_sampler.hip

Both backends can coexist in the same CMakeLists.txt. Each produces a separate static library target.

11. The Role of the VM

The SVM is central: both CPU and GPU run the same abstract machine.

CPU SVM

GPU SVM

For low k (1–3), the per-shot work is tiny → GPU wins on throughput.
For high k, CPU SIMD advantage would grow — but QEC circuits keep k low.

12. Potential Optimizations

Current state: mechanical port, not optimized for either GPU architecture.

Shared Memory for Bytecode

Currently bytecode is read from global memory. Loading it into shared memory at block start could reduce latency.

Warp-Level Reduction

Replace shared-memory tree reduction with __shfl_down_sync() / warp intrinsics for the intra-warp phase.

Memory Coalescing

Measurement record writes are per-thread, non-coalesced. An SoA layout could improve memory throughput.

Persistent Kernel Pattern

One thread/block runs multiple shots in a loop (already used for global_coop mode). Extend to all modes.

Mixed Precision

PRNG uses double for uniform(). Could use float for noise threshold comparisons where precision allows.

Multi-GPU Scaling

Partition total_shots across devices. No inter-device communication needed — just sum counters.
