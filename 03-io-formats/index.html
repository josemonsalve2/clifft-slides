Input Format: Stim Circuit Text

All three input paths consume the same underlying format: Stim circuit text.

From file

# Python: read file, then compile
circuit_text = open("surface_code.stim").read()
program = clifft.compile(circuit_text)
# GPU CLI: pass file path directly
./run_msc_hip --circuit surface_code.stim

Inline string

# Python: inline circuit string
program = clifft.compile("""
H 0
CNOT 0 1
M 0 1
""")

The Stim format is line-oriented: one instruction per line, whitespace-separated targets.

The Stim Circuit Format

Line-by-line syntax deep dive

Basic Syntax

INSTRUCTION target1 target2 ...
INSTRUCTION(parameter) target1 target2 ...

# Lines starting with # are comments
# Empty lines are ignored
INSTRUCTION
+
targets (qubit indices)
+
optional (param)

Examples

H 0              # Hadamard on qubit 0
CNOT 0 1         # CNOT: control=0, target=1
M 0 1 2          # Measure qubits 0, 1, 2
DEPOLARIZE1(0.001) 0 1 2

Multi-target shorthand

# Apply H to qubits 0, 1, 2, 3
H 0 1 2 3

# CNOT pairs: (0,1) and (2,3)
CNOT 0 1 2 3

XPauli-X (bit flip)σxX 3

S_DAGS-adjointdiag(1, −i)S_DAG 0

Measurement & Reset Instructions

Measurement

InstructionBasisExample
MZ-basisM 0 1 2
MXX-basisMX 0

Reset

InstructionActionExample

Measurement Record References

# rec[-1] = most recent measurement result
# rec[-2] = second most recent, etc.
M 0 1 2
DETECTOR rec[-1] rec[-2]   # parity of last two measurements

Noise Instructions

InstructionChannelExample
DEPOLARIZE1(p) Single-qubit depolarizing: each of X,Y,Z with prob p/3 DEPOLARIZE1(0.001) 0 1 2
DEPOLARIZE2(p) Two-qubit depolarizing: 15 Pauli pairs with prob p/15 DEPOLARIZE2(0.01) 0 1
X_ERROR(p) Bit-flip error with probability p X_ERROR(0.001) 0
Z_ERROR(p) Phase-flip error with probability p Z_ERROR(0.001) 0
PAULI_CHANNEL_1(px,py,pz) General 1-qubit Pauli channel PAULI_CHANNEL_1(0.01,0.005,0.005) 0

Noise channels are sampled stochastically during simulation. Use clifft.compile(text, hir_passes=None) to keep noise, or the RemoveNoisePass to strip it.

Control Flow & Layer Structure

REPEAT block

# Repeat a block N times
REPEAT 100 {
    H 0
    CNOT 0 1
    DEPOLARIZE1(0.001) 0 1
    M 0 1
    DETECTOR rec[-1] rec[-3]
    DETECTOR rec[-2] rec[-4]
    R 0 1
    TICK
}

Essential for QEC: encode N rounds of syndrome extraction in a compact loop.

TICK — Layer separator

# TICK separates time layers
H 0 1 2
TICK
CNOT 0 1 2 3
TICK
M 0 1 2 3

TICK does not affect simulation semantics. It marks circuit layer boundaries for visualization and analysis.

Detectors & Observables

DETECTOR

# Parity check on measurement records
# Should be deterministic in noiseless case
DETECTOR rec[-1] rec[-5]
DETECTOR rec[-2] rec[-6]

# With coordinate annotations
DETECTOR(1, 2, 0) rec[-1] rec[-5]

Detectors define syndrome bits. Under noise, a detection event (bit = 1) flags an error.

OBSERVABLE_INCLUDE

# Track logical observable
# Observable index in parens
OBSERVABLE_INCLUDE(0) rec[-1]
OBSERVABLE_INCLUDE(0) rec[-3]

# Multiple observables
OBSERVABLE_INCLUDE(1) rec[-2]

Observables track logical qubit values. A flip (bit = 1) means a logical error.

Measurements
→
rec[-N] references
→
DETECTOR (parity check)
→
Detection events (0/1)

03

Clifft Extensions to Stim

Non-Clifford gates that make Clifft special

Non-Clifford Gate Extensions

Standard Stim only handles Clifford gates. Clifft extends the format with non-Clifford gates that increase the active dimension k (peak_rank).

InstructionGateMatrixEffect on kExample
T T gate (pi/8) diag(1, eiπ/4) +1 per T T 0
T_DAG T adjoint diag(1, e−iπ/4) +1 per T_DAG T_DAG 0
RZ(angle) Arbitrary Z-rotation diag(1, eiθ) +1 per RZ RZ(0.785) 0
S Phase gate diag(1, i) 0 (Clifford) S 0

Why this matters

  • SVM cost scales as O(2k) per shot
  • Clifford-only circuits: k = 0 (constant time)
  • Each non-Clifford gate may increase k by 1
  • Optimization passes can reduce k via gate fusion

S gate: the exception

  • S is Clifford — absorbed into Pauli frame tracking
  • Does not increase active dimension
  • Handled by Heisenberg representation, not SVM

Clifft vs Stim: Gate Support Comparison

Stim (Clifford only)
H, X, Y, Z, S, S_DAG, SQRT_X/Y,
CNOT, CZ, SWAP, ISWAP, M, R
↓ superset
Clifft (near-Clifford)
All Stim gates + T, T_DAG, RZ(θ)
# This circuit is valid Clifft,
# but NOT valid Stim:

H 0
T 0          # non-Clifford!
CNOT 0 1
T_DAG 1      # non-Clifford!
RZ(0.123) 2  # non-Clifford!
M 0 1 2
# Check peak rank after compile
prog = clifft.compile(circuit_text)
print(f"Peak rank: {prog.peak_rank}")
# Peak rank: 3  (three non-Clifford gates)

Compilation Examples

import clifft

# --- Basic compile ---
circuit = """
H 0
CNOT 0 1
DEPOLARIZE1(0.001) 0 1
M 0 1
DETECTOR rec[-1] rec[-2]
OBSERVABLE_INCLUDE(0) rec[-1]
"""
program = clifft.compile(circuit)


# --- With syndrome normalization (auto-detect reference parities) ---
program = clifft.compile(circuit, normalize_syndromes=True)


# --- With all-detector postselection (discard shots with detection events) ---
program = clifft.compile(
    circuit,
    normalize_syndromes=True,
    postselection_mask=[1] * program.num_detectors,  # postselect all detectors
)


# --- Skip optimization passes ---
program = clifft.compile(circuit, hir_passes=None, bytecode_passes=None)


# --- Custom pass pipeline ---
hpm = clifft.default_hir_pass_manager()  # get default, modify as needed
bpm = clifft.default_bytecode_pass_manager()
program = clifft.compile(circuit, hir_passes=hpm, bytecode_passes=bpm)

Available Optimization Passes

HIR Passes

PassEffect
RemoveNoisePassStrip noise channels
DropNonUnitaryPassDrop measurements/resets
ExpandTPassDecompose T gates
ExpandRotPassDecompose rotations
MultiGatePassMulti-gate fusion
NoiseBlockPassNoise block optimization
SwapMeasPassSwap-measure reordering

Bytecode Passes

PassEffect
PeepholeFusionPassFuse adjacent ops
SingleAxisFusionPassMerge single-axis rotations
StatevectorSqueezePassReduce active dimension k

Passes reduce peak_rank and instruction count, directly improving simulation speed.

Lower-Level Pipeline Access

import clifft

circuit_text = open("circuit.stim").read()

# Step 1: Parse to AST
circuit = clifft.parse(circuit_text)
# or: circuit = clifft.parse_file("circuit.stim")

# Step 2: Trace to HIR (High-level Intermediate Representation)
hir = clifft.trace(circuit)

# Step 3: Run HIR optimization passes
hpm = clifft.default_hir_pass_manager()
hpm.run(hir)  # mutates hir in-place

# Step 4: Compute reference syndrome (noiseless parities)
ref = clifft.compute_reference_syndrome(hir)
# ref.detectors: list of expected detector values
# ref.observables: list of expected observable values

# Step 5: Lower HIR to bytecode
program = clifft.lower(
    hir,
    postselection_mask=[1] * len(ref.detectors),
    expected_detectors=ref.detectors,
    expected_observables=ref.observables,
)

# Step 6: Run bytecode optimization passes
bpm = clifft.default_bytecode_pass_manager()
bpm.run(program)  # mutates program in-place

clifft.sample()

import clifft
import numpy as np

circuit = """
H 0
CNOT 0 1
DEPOLARIZE1(0.01) 0 1
M 0 1
DETECTOR rec[-1] rec[-2]
OBSERVABLE_INCLUDE(0) rec[-1]
"""
program = clifft.compile(circuit, normalize_syndromes=True)

# --- Standard sampling ---
result = clifft.sample(program, shots=10000, seed=42)

print(result)
# SampleResult(shots=10000, measurements=2)

print(result.measurements.shape)  # (10000, 2)  - uint8
print(result.detectors.shape)     # (10000, 1)  - uint8
print(result.observables.shape)   # (10000, 1)  - uint8

# Fraction of shots with detection events
detection_rate = np.mean(result.detectors)
print(f"Detection rate: {detection_rate:.4f}")

# Tuple unpacking (backward compatibility)
m, d, o = clifft.sample(program, shots=1000)

clifft.sample_survivors()

# --- Postselection: discard shots with any detection event ---
program = clifft.compile(
    circuit,
    normalize_syndromes=True,
    postselection_mask=[1] * 1,  # postselect on all detectors
)

result = clifft.sample_survivors(program, shots=100000, seed=42)

print(f"Total shots attempted:  {result.total_shots}")
print(f"Shots surviving:        {result.passed_shots}")
print(f"Discarded:              {result.discards}")
print(f"Logical errors:         {result.logical_errors}")
print(f"Survival rate:          {result.passed_shots / result.total_shots:.4f}")
print(f"Error rate (survivors): {result.logical_errors / result.passed_shots:.6f}")

# Per-observable error counts
print(f"Observable errors:      {result.observable_ones}")

# Measurements array only contains surviving shots
print(f"Measurements shape:     {result.measurements.shape}")
# (passed_shots, num_measurements)

Exact probabilities

# Born probabilities (unitary circuits)
prog = clifft.compile("H 0\nCNOT 0 1")
probs = clifft.basis_probabilities(
    prog, ["00", "01", "10", "11"]
)
# array([0.5, 0.0, 0.0, 0.5])

# Log-space for tiny probabilities
log_p = clifft.basis_probabilities(
    prog, "00", return_log=True
)

# Measurement record probabilities
prog2 = clifft.compile("H 0\nM 0")
p = clifft.record_probabilities(
    prog2, ["0", "1"]
)
# array([0.5, 0.5])

Statevector Access

# Get amplitude of a specific basis state
prog = clifft.compile("H 0\nCNOT 0 1")

amp_00 = clifft.get_statevector(prog, 0)  # basis index 0 = |00>
print(f"|00> amplitude: {amp_00}")         # (0.707...+0j)

amp_01 = clifft.get_statevector(prog, 1)  # basis index 1 = |01>
print(f"|01> amplitude: {amp_01}")         # 0j

amp_10 = clifft.get_statevector(prog, 2)  # basis index 2 = |10>
print(f"|10> amplitude: {amp_10}")         # 0j

amp_11 = clifft.get_statevector(prog, 3)  # basis index 3 = |11>
print(f"|11> amplitude: {amp_11}")         # (0.707...+0j)

Note: get_statevector retrieves one amplitude at a time. For unitary circuits, use basis_probabilities for batch probability queries. Cost scales with peak_rank k, not total qubits n.

Noise Site Probabilities — Detailed Example

import clifft
import numpy as np

circuit = """
H 0
CNOT 0 1
DEPOLARIZE1(0.001) 0 1
DEPOLARIZE2(0.01) 0 1
X_ERROR(0.005) 0
Z_ERROR(0.003) 1
M 0 1
"""
program = clifft.compile(circuit)

# Each noise instruction creates one or more noise sites
probs = program.noise_site_probabilities
print(f"Number of noise sites: {len(probs)}")
print(f"Probabilities: {probs}")
# e.g. array([0.001, 0.001, 0.01, 0.005, 0.003])

# Use for importance sampling analysis
total_fault_rate = 1.0 - np.prod(1.0 - probs)
print(f"Prob of at least one fault: {total_fault_rate:.6f}")

Output Format Details

Record arrays, aggregated stats, and data shapes

Record Types — Bit Arrays

Measurement Records

shape: (shots, num_measurements)
dtype: uint8 (0 or 1)
shot 0: [ 0 1 0 1 1 0 ]
shot 1: [ 1 0 0 1 0 1 ]
shot 2: [ 0 0 1 1 1 0 ]
... N shots ...

Raw measurement outcomes. Position i = i-th measurement instruction in circuit order.

Detector Records

shape: (shots, num_detectors)
dtype: uint8 (0 or 1)
shot 0: [ 0 0 0 0 ] no errors
shot 1: [ 0 1 0 0 ] detection event!
shot 2: [ 0 0 0 0 ] no errors
... N shots ...

0 = matches noiseless reference. 1 = detection event (error syndrome triggered).

Python vs GPU CLI: Output Comparison

Python API (CPU)

  • Per-shot record arrays (numpy)
  • Full measurement, detector, observable records
  • Caller does aggregation
  • Can inspect individual shots
  • Memory: O(shots * records)

  • Aggregated statistics (JSON)
  • No per-shot records (GPU-side reduction)
  • Pre-computed rates and counts
  • Constant memory output
  • Memory: O(1) output

Quick API Cheat Sheet

Compile & Sample (CPU)

import clifft

# Compile
prog = clifft.compile(
    open("circuit.stim").read(),
    normalize_syndromes=True,
)

# Sample
r = clifft.sample(prog, shots=10000)
r.measurements  # [S, M] uint8
r.detectors     # [S, D] uint8
r.observables   # [S, O] uint8

# Survivors
r = clifft.sample_survivors(prog, 100000)
r.passed_shots, r.logical_errors

# Importance sampling
r = clifft.sample_k(prog, 10000, k=2)

# Exact probabilities
p = clifft.basis_probabilities(
    prog, ["00", "11"])
p = clifft.record_probabilities(
    prog, ["01", "10"])
