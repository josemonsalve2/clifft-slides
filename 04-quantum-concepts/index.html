Why Complex Numbers?

Phase matters — it creates interference

Amplitudes are complex: a + bi

Two paths to the same outcome:

Path A amplitude: +0.5

Path B amplitude: -0.5

+0.5 + (-0.5) = 0

Destructive interference! Probability = 0

Contrast: Classical Probabilities

Two paths to the same outcome:

Path A probability: 0.25

Path B probability: 0.25

0.25 + 0.25 = 0.50

Always adds up! No cancellation.

GPU analogy: Think of amplitudes like wave amplitudes in a physics sim — they can add constructively or destructively. This is the source of quantum computational power.

Entanglement

Correlations with no classical explanation

Separable (NOT entangled)

What is a quantum gate?

How to simulate quantum circuits in polynomial time

What is the Clifford Group?

Definition

The set of all unitary operations that map Pauli operators to Pauli operators under conjugation:

C P C† = P'
where P, P' are Pauli operators

In other words: Clifford gates "shuffle" Pauli matrices around but keep them as Pauli matrices.

Generated by just 3 gates

Controlled-NOT

Every Clifford gate can be decomposed into a sequence of H, S, and CNOT gates. That's the entire infinite group from just 3 generators!

Why Clifford Circuits are Classically Simulatable

The Gottesman-Knill Theorem

A quantum circuit can be simulated classically in polynomial time if it uses only:

  1. Clifford gates (H, S, CNOT, and compositions)
  2. Computational basis measurements
  3. Classical feed-forward (conditional gates)
  4. Starting from a computational basis state

Complexity

Per gate: O(n) row ops

Total for L gates: O(n·L)

Memory: O(n²)

Compare: full state vector needs O(2n) memory and O(2n) per gate!

Stim (by Google) implements exactly this. It can simulate millions of Clifford qubits in seconds. But it cannot handle T gates at all — that's where Clifft comes in.

What Breaks Clifford Simulability?

T gates and "magic"

T Gate
10 0eiπ/4
T does NOT map Paulis to Paulis!
T X T† = &frac1;√2 (X + Y) — not a Pauli!

The T gate adds "magic"

  • A state produced by T gates is called a "magic state"
  • Magic states cannot be described by stabilizers alone
  • Each T gate can double the simulation cost
  • k T gates → up to 2k stabilizer terms needed

Clifford + T = universal quantum computing.
You need non-Clifford gates for quantum advantage, but they're exactly what makes simulation hard!

Quantum Error Correction

Protecting quantum information from noise

Why We Need Quantum Error Correction

Physical qubits are noisy

  • Decoherence: qubits lose their quantum state over time
  • Gate errors: ~0.1-1% error per operation
  • Measurement errors: ~0.5-5% chance of wrong readout
  • Crosstalk: operations on one qubit affect neighbors

A useful quantum computation needs millions of gates. With 0.1% error per gate, after 1000 gates you've lost all information!

Classical Error Correction Analogy

Repetition Code

Encode: 0 → 000   1 → 111

If one bit flips:

010 → majority vote → 000 ✓

Can correct any single-bit error

Parity Checks

Check 1: bit1 ⊕ bit2 = 0 (should match)

Check 2: bit2 ⊕ bit3 = 0 (should match)

If bit 2 flips:

Check 1: 0 ⊕ 1 = 1 ← error detected!

Check 2: 1 ⊕ 0 = 1 ← error detected!

Both checks fire → error is on bit 2

Quantum version: Same idea, but we use stabilizer measurements instead of parity checks, and we can detect both bit-flip (X) AND phase-flip (Z) errors.

Key Concepts

  • Data qubits hold the encoded quantum information
  • Ancilla qubits (X and Z stabilizers) are measured to detect errors
  • X stabilizers detect Z (phase-flip) errors
  • Z stabilizers detect X (bit-flip) errors

Code distance d: The code can correct up to ⌊(d-1)/2⌋ errors. d=3 → correct 1 error. d=5 → correct 2 errors. Larger d = more physical qubits but better protection.

Syndrome Extraction

Measuring stabilizers to detect errors without disturbing data

Each stabilizer measurement cycle:

1. Prepare ancilla in 0 (for Z-type) or + (for X-type)
2. Apply CNOT gates between ancilla and neighboring data qubits
3. Measure ancilla → get syndrome bit (0 = no error, 1 = error detected)
4. Repeat for all stabilizers → syndrome = pattern of detection events

Critical property: The stabilizer measurements are all Clifford operations (CNOT + measurement). This is why Stim can simulate QEC so efficiently! The entire syndrome extraction circuit is Clifford.

Detectors and Observables

How Stim/Clifft track error detection

DETECTOR

A parity check: XOR of measurement results that should be deterministic (always 0 in the absence of errors).

DETECTOR rec[-1] rec[-5]

"XOR of the last measurement and the one from 5 ago should be 0"

  • If detector fires (result = 1) → an error happened nearby
  • Pattern of detector firings = error syndrome
  • Feed to a decoder to identify the error

OBSERVABLE

Tracks the logical qubit value through the circuit. A parity of measurements that reveals the encoded information.

OBSERVABLE_INCLUDE(0) rec[-1]

"This measurement contributes to logical observable 0"

  • Logical error = observable flipped by undetected errors
  • The decoder's job: prevent observable flips
  • Logical error rate is the key metric

Noise Models

How real quantum hardware fails

Depolarizing Noise

The most common noise model

Single-qubit depolarizing (probability p)

With probability 1-p: nothing happens

With probability p/3: apply X (bit flip)

With probability p/3: apply Y (bit+phase flip)

With probability p/3: apply Z (phase flip)

The "leaky pipe" model

Perfect Gate
Random Pauli
Every noisy gate = ideal gate + random Pauli error

Why Paulis? Any single-qubit error can be decomposed into a probabilistic mixture of I, X, Y, Z. Since Clifford circuits map Paulis to Paulis, Pauli noise plays nicely with the stabilizer formalism.

Types of Noise Channels

Bit-flip (X error)

0 → 1 with probability p

Like classical bit corruption

Stim: X_ERROR(p) target

Phase-flip (Z error)

+ → - with probability p

No classical analogue!

Stim: Z_ERROR(p) target

Two-qubit depolarizing

15 possible Pauli pairs: IX, IY, IZ, XI, XX, ... ZZ

Each with probability p/15

Stim: DEPOLARIZE2(p) q1 q2

Measurement noise

Readout result flips with probability p

The qubit state is unchanged!

Stim: MR(p) target or M(p) target

Noise in a Circuit: The Full Picture

q0: — H — noise — • — noise — M —
q1: ————————————— ⊕ —————————————
After each gate layer: insert noise. Measurement also has its own noise.

In Stim/Clifft: Noise is just Pauli errors inserted after gates. Since Pauli errors commute nicely through Clifford circuits, the simulator can track them efficiently in the Pauli frame — noise doesn't increase k!

Section 7

Stim — The Foundation

Google's fast Clifford simulator that Clifft builds on

What is Stim?

Google Research's Clifford simulator

  • Written in C++ with Python bindings
  • Simulates Clifford circuits insanely fast
  • Core of Google's QEC research pipeline
  • Can sample from circuits with millions of qubits

Key capability: Stim can do ~1 billion Clifford gates per second on a single core. It uses SIMD-optimized tableau operations.

Stim's two simulation modes

Tableau Simulation

Full state tracking via stabilizer tableau. Exact amplitudes. Useful for small circuits and verification.

Frame Simulation

Track Pauli frames (error propagation). Blazing fast. Only gives measurement results, not full state. Used for sampling in QEC.

The .stim Circuit Format

One instruction per line, simple and readable

# Bell state creation + measurement
H 0                   # Hadamard on qubit 0
CNOT 0 1              # CNOT with control=0, target=1
DEPOLARIZE2(0.001) 0 1 # 0.1% two-qubit noise
M 0 1                 # Measure qubits 0 and 1
DETECTOR rec[-1] rec[-2] # Parity check
OBSERVABLE_INCLUDE(0) rec[-1]

Instruction types

  • GATE target — apply gate
  • NOISE(p) target — apply noise
  • M target — measure (destructive)
  • MR target — measure + reset

Meta instructions

  • DETECTOR rec[...] — parity check
  • OBSERVABLE_INCLUDE — logical qubit
  • REPEAT n { ... } — loop
  • # comment — comments

Stim QEC Circuit Example

Repetition code with 3 rounds

# d=3 repetition code
R 0 1 2 3 4            # Reset 3 data + 2 ancilla qubits

REPEAT 3 {
    CNOT 0 3  1 4       # First round of CNOTs
