Step-by-Step: How the Stim Tableau Absorbs Gates

1 H 0 — Absorbed

Hadamard on qubit 0 is a Clifford gate. The frontend applies it to the internal Stim tableau. No HIR op emitted. The tableau now maps X₀↔Z₀.

2 CNOT 0 1 — Absorbed

CNOT is Clifford. Applied to the tableau: X₁→X₀X₁, Z₀→Z₀Z₁. No HIR op emitted. The tableau now tracks the accumulated effect of H+CNOT.

3 S 2 — Absorbed

S gate is Clifford. Applied to tableau: Z₂→X₂Z₂. No HIR op emitted.

4 T 2 — EMITTED ✦

T gate is non-Clifford — cannot be absorbed! The frontend "rewinds" T through the current tableau to determine its Pauli mask, then emits: HirOp::T_GATE(mask=Z₂). The mask reflects S's effect (Z₂ instead of raw Z₂).

5 DEPOLARIZE1 — EMITTED ✦

Noise can't be absorbed (it's stochastic). Each qubit's noise channel is rewound through the tableau → Pauli masks capture the Clifford context. E.g., q0's X-error becomes X⊗I⊗I after the H+CNOT transformation.

6 M 0 1 2 — EMITTED ✦

Measurements are rewound through the tableau. M(q0) → measures the Pauli X⊗I⊗I (because H transformed Z→X, then CNOT spread it). M(q1) → X⊗X⊗I. M(q2) → I⊗I⊗Y (because S transformed Z→Y-ish).

The Key Insight: Why Absorption Works

1. Compile Time

The Stim Tableau (2n × 2n binary matrix) tracks the accumulated Clifford circuit. Each Clifford gate = a row operation on the matrix. Cost: O(n) per gate.

2. Rewind Non-Cliffords

When a T gate, measurement, or noise is encountered, the tableau "rewinds" it — computing the Pauli mask that captures the Clifford context. This mask is stored in the HIR.

3. Runtime

The SVM uses a lightweight Pauli frame (2n bits) instead of the full tableau. Clifford gates = O(1) bit flips. The Pauli masks from compilation tell each op what to do.

When T(2) arrives: The frontend looks up qubit 2's current Pauli in the tableau and emits HirOp::T_GATE with a Pauli mask derived from the current X₂ and Z₂ columns. The S gate's effect (Z₂ column now has X₂=1) is baked into the mask — no separate S instruction will ever appear in the HIR or bytecode.
