What is Clifft?

Clifft at a Glance

  • Fast exact simulator for near-Clifford quantum circuits
  • Maintained by Unitary Foundation (Brad Chase)
  • Primary target: QEC circuit simulation
  • Written in C++20, header-only core, pybind11 Python bindings
  • ISA-optimized: AVX-512, AVX2, scalar backends (runtime CPUID)

Why "Near-Clifford"?

A QEC circuit with n qubits may have thousands of gates, but the vast majority are Clifford gates (H, S, CNOT, CZ) — classically simulable in O(n²) via the Gottesman-Knill theorem.

Only a handful of non-Clifford gates (T, arbitrary rotations) require exponential-cost statevector operations.

Clifft exploits this asymmetry.

The Key Insight

Clifford Gates
H, S, CNOT, CZ, SWAP
O(n) — Pauli frame update only
Non-Clifford
T, Rz(θ)
O(2^k) — statevector

Factored State Representation

|ψ⟩ = γ · U_C · P · ( |φ⟩_A ⊗ |0⟩_D )
  • γ — global scalar (phase + normalization)
  • U_C — Clifford unitary (absorbed into tableau)
  • P — Pauli frame (p_x, p_z bitmasks)
  • |φ⟩_A — active statevector: 2^k amplitudes
  • |0⟩_D — dormant qubits (not stored)
  • k = active rank (typically 1–3 for QEC)

Each Stage at a Glance

1

Parser

Reads .stim format with extensions: T, T_DAG, S, S_DAG, RZ(θ). Produces a Circuit object (gate list + qubit metadata).

2

Frontend (Tableau Trace)

Drives Stim's TableauSimulator. Cliffords are absorbed. Non-Clifford gates and measurements emit HeisenbergOps with rewound Pauli masks into the HirModule.

3

Optimizer

11 registered passes (4 HIR-level, 7 bytecode-level). Fuse gates, minimize active rank k, eliminate noise, peephole optimize. Single pass manager drives all.

4

Backend Compiler (AOT)

Localizes global qubit topology into virtual-axis operations. Emits 32-byte aligned bytecode instructions + ConstantPool (Pauli masks, fused unitaries, noise channels).

5

SVM Execution

Iterates bytecode, dispatches to ISA-specific kernels. Each shot: reset state → execute → collect measurements/detectors/observables. PRNG: Xoshiro256++.

HIR — Heisenberg Intermediate Representation

Key Design Principle

Clifford gates never appear in the HIR. They are fully absorbed into Stim's tableau during the frontend trace. Only non-Clifford operations and measurements survive.

Each HeisenbergOp carries:

  • PauliMaskHandle — reference into arena
  • Weight (complex coefficient)
  • Axis indices (virtual qubit positions)

Side Tables on HirModule

  • noise_sites — NoiseSite channels
  • detector_targets — measurement record indices
  • observable_targets — observable accumulators
  • readout_noise — bit-flip entries

Frontend Transformation

Algorithm: Tableau-Driven Trace

  1. Initialize Stim TableauSimulator with identity tableau
  2. For each gate in the circuit:
    • Clifford (H, S, CNOT, CZ, SWAP): Apply to simulator — absorbed into tableau, no HIR op emitted
    • T / T_DAG: Extract rewound Z-observable from inv_state.zs[q], emit T_GATE HeisenbergOp with Pauli mask
    • Measurement: Extract rewound observable, emit MEASURE
    • Classical feedback (CX/CZ with rec targets): Extract rewound Pauli, emit CONDITIONAL_PAULI
  3. Return HirModule with all emitted operations
Clifford gates → Stim Tableau × T gate on qubit q → Extract Z_q from
inv_state.zs[q] → HeisenbergOp
(T_GATE, mask, weight)

Optimizer Passes

HIR Passes (pre-compilation)

PeepholeFusionPass [default: ON]
Local gate fusion — merges adjacent same-axis ops into fused U2/U4 unitaries
StatevectorSqueezePass [default: ON]
Minimizes peak active rank k by reordering operations — critical for keeping 2k small
RemoveNoisePass [default: OFF]
Strips all noise channels for noiseless simulation
DropNonUnitaryPass [default: OFF]
Strips measurements for probability computation mode

Bytecode Passes (post-compilation)

NoiseBlockPass [default: ON]
Groups contiguous noise sites into OP_NOISE_BLOCK
MultiGatePass [default: ON]
Fuses consecutive CNOT/CZ into OP_ARRAY_MULTI_CNOT/CZ
ExpandTPass [default: ON]
Fuses OP_EXPAND + OP_ARRAY_T into single OP_EXPAND_T
ExpandRotPass [default: ON]
Fuses OP_EXPAND + OP_ARRAY_ROT into OP_EXPAND_ROT
SwapMeasPass [default: ON]
Fuses OP_ARRAY_SWAP + OP_MEAS into OP_SWAP_MEAS
TileAxisFusionPass [default: ON]
Fuses gates across tile boundaries
SingleAxisFusionPass [default: ON]
Fuses consecutive same-axis operations

Instruction Layout: 32 bytes exactly

2 instructions per 64-byte L1 cache line. Fields: opcode (1B), flags (1B), axis_1/axis_2 (2B each), 24B payload union (math / classical / pauli / multi_gate / u2 / u4).

Memory Per Shot (CPU)

  • v[]: 2peak_rank × 16 bytes
  • p_x, p_z: 2 × ceil(n/64) × 8 bytes
  • meas_record: num_meas bits
  • PRNG state: 32 bytes (4 × uint64)
  • For k=3, n=100: ~160 bytes total

How Operations Act on the State

Clifford (Frame Opcodes) — O(n)

Only update p_x and p_z bitmasks. Never touch v[].

  • OP_FRAME_H(q): swap p_x[q] ↔ p_z[q]
  • OP_FRAME_S(q): p_x[q] ^= p_z[q]
  • OP_FRAME_CNOT(c,t): p_x[t] ^= p_x[c]; p_z[c] ^= p_z[t]
  • OP_FRAME_CZ(a,b): p_x[a] ^= p_z[b]; p_x[b] ^= p_z[a]

Cost: a few XOR operations on 64-bit words

Non-Clifford (Array Opcodes) — O(2k)

Must loop over all 2k amplitudes in v[].

  • OP_ARRAY_T(axis): v[i] *= eiπ/4 where bit axis is set
  • OP_ARRAY_H(axis): butterfly on pairs (Hadamard)
  • OP_ARRAY_CNOT(c,t): swap v[] pairs by bitmask
  • OP_EXPAND(axis): k → k+1, γ /= √2

Cost: proportional to array size 2k

Measurement (k may decrease)

Dormant Static deterministic from p_x Dormant Random random pivot, k unchanged Active Diagonal Z-filter, k→k-1 Active Interfere X-fold, k→k-1

Execution Loop & ISA Backends

Main Execution Loop (per shot)

state.reset();          // v = |0>, p_x = p_z = 0
for (auto& instr : bytecode) {
    switch (instr.opcode) {
        case OP_FRAME_H:
            // swap p_x[axis] <-> p_z[axis]
            break;
        case OP_EXPAND_T:
            // k -> k+1, apply T phase
            break;
        case OP_MEAS_ACTIVE_DIAGONAL:
            // sample, collapse, k -> k-1
            break;
        case OP_NOISE:
            // roll RNG, maybe apply Pauli
            break;
        // ... ~35 opcodes total
    }
}
// collect measurements, detectors, observables

ISA Detection & Dispatch

Runtime CPUID selects the best available kernel implementation:

AVX-512 8 complex doubles per SIMD op
AVX2 4 complex doubles per SIMD op
Scalar Fallback, no SIMD

PRNG: Xoshiro256++

  • Period: 2256 - 1
  • State: 32 bytes (vs MT19937's 2504 bytes)
  • Deterministic cross-platform (pure bitwise)
  • Seeded via SplitMix64 expansion
  • Guarantees identical sequences: GCC = Clang = MSVC

The Sampling Loop

.stim file → Compile Once → CompiledModule
↓
REPEAT N SHOTS
Reset State → Execute Bytecode → Collect Record
v[] = |0⟩, p_x = p_z = 0   |   iterate opcodes   |   meas + detectors + observables
↓
Aggregate Results → Error rates, detection events, observable flips

Key optimization: Compilation cost is amortized. State reset() reuses allocation — only zeroes the arrays. PRNG streams forward naturally between shots (no reseed).

GPU Backends

CUDA & AMD HIP

Why GPU? Embarrassingly Parallel

The Parallelism Opportunity

  • Each shot is completely independent
  • Same bytecode, different PRNG stream
  • No inter-shot communication needed
  • Per-shot state is tiny (k ≤ 4 ⇒ fits in registers)
  • Perfect for GPU: thousands of threads, each running full SVM

One GPU Thread = One Shot

Each CUDA/HIP thread independently:

  1. Initializes local state (v[], p_x, p_z, PRNG)
  2. Interprets full bytecode program
  3. Writes results to global memory

    GpuProgram prog,
    uint64_t base_seed,
    uint64_t num_shots,
    BlockCounts* block_counts)

    for (uint32_t ip = 0;
         ip < prog.num_instrs; ip++) {
        auto& ins = prog.instrs[ip];
        switch (ins.opcode) {
            // ... same opcodes as CPU SVM
        }
    }

    // 4. Reduce results (atomics)
    atomicAdd(&block_counts->passed, 1);

Why Billions of Shots?

To measure a logical error rate of 10-9 with reasonable statistics, you need at least ~10× more trials than 1/pL.

At 1.2M shots/s on GPU:

  • 1011 shots ≈ ~24 hours
  • Expect ~100 logical errors → meaningful statistics

Importance Sampling

sample_k(N, k): Force exactly k noise faults per shot to find rare failure modes faster. Trades unbiased sampling for targeted exploration of the error space.

Runtime Complexity

kPeak active rank (non-Clifford dimension)

The 2k factor dominates. For QEC circuits with k ≤ 3, the inner loop touches only 8 amplitudes — trivially fits in registers.

What Makes Clifft Fast?

  1. Separation of concerns: Clifford ops are O(1) frame updates, not O(2k) array ops
  2. StatevectorSqueezePass: Minimizes peak k by reordering ops
  3. Peephole fusion: Merges consecutive array ops into fused U2/U4
  4. 32-byte instructions: 2 per L1 cache line
  5. SIMD kernels: AVX-512 processes 8 amplitudes/cycle
  6. GPU parallelism: Thousands of independent shots
  7. Compact PRNG: 32-byte Xoshiro256++ state per shot
