01

Optimizer Overview

Two tiers, eleven passes, one goal: minimize O(2^k) runtime

Two-Tier Architecture

HIR Passes (Pre-Lowering)

Operate on HirModule -- the high-level Heisenberg IR

  • Algebraic: cancel/fuse T gates
  • Scheduling: reorder ops to minimize peak k
  • Filtering: strip noise or non-unitary ops
class HirPass {
  public:
    virtual void run(HirModule& hir) = 0;
    virtual ~HirPass() = default;
};

Bytecode Passes (Post-Lowering)

Operate on CompiledModule -- the finalized 32-byte instruction stream

  • Instruction fusion: merge adjacent ops
  • CISC synthesis: precompute fused matrices
  • Reduce array sweeps (each costs O(2^k))
class BytecodePass {
  public:
    virtual void run(CompiledModule& module) = 0;
    virtual ~BytecodePass() = default;
};

02

PeepholeFusionPass

HIR DEFAULT ON

Algebraic T/T_dag cancellation and fusion via symplectic commutation

Peephole: What It Does

Purpose

Scans the HIR for pairs of T/T_dag gates and PHASE_ROTATION ops acting on the same virtual Pauli axis. Uses the symplectic inner product to commute past non-interfering operations and find matchable pairs.

Three Transformations

  1. Cancellation: T(P) + T_dag(P) = Identity -- delete both
  2. S-Absorption: T(P) + T(P) = S(P) -- absorb into tableau
  3. Rotation Folding: ROT(P, a) + ROT(P, b) = ROT(P, a+b)

Why This Matters

Every surviving T gate forces k -> k+1 (doubles the amplitude array). Cancelling even one T gate halves peak memory and runtime.

Reports stats: cancellations() and fusions()

Peephole: T + T_dag Cancellation

BEFORE:  HIR Operation Stream
+-------+-------+-------+-------+-------+-------+
| T(P1) | M(Q)  | N(R)  | T_dag | M(S)  | T(P2) |
|       | commu-| commu-| (P1)  |       |       |
|       | tes   | tes   |       |       |       |
+-------+-------+-------+-------+-------+-------+
         ^       ^
         |       |
    Both commute with P1 (symplectic inner product = 0)
    so the peephole can "see through" them

AFTER:   T + T_dag on same axis P1 --> Identity (cancelled)
+-------+-------+-------+-------+
| M(Q)  | N(R)  | M(S)  | T(P2) |
+-------+-------+-------+-------+

Result: k reduced by 1.  If k was 3, runtime drops from O(8) to O(4)

Commutation Check (commutation.h)

inline bool anti_commute(MaskView x1, MaskView z1, MaskView x2, MaskView z2) {
    int parity = 0;
    for (uint32_t i = 0; i < x1.num_words(); ++i)
        parity += std::popcount((x1.words[i] & z2.words[i]) ^
                                (z1.words[i] & x2.words[i]));
    return (parity & 1) != 0;  // true = anti-commutes = BLOCKED
}

Peephole: T + T Fusion (S-Absorption)

BEFORE:                                 AFTER:
+-------+-------+-------+             +-------+-------+
| T(P1) | M(Q)  | T(P1) |             | M(Q') |  ...  |
|       | commu-|       |             |       |       |
|       | tes   |       |             |       |       |
+-------+-------+-------+             +-------+-------+

T + T = S gate on axis P1
S is Clifford --> absorbed into the Stim Tableau, never appears in bytecode

The S-absorption propagates through ALL downstream ops:
  - Conjugates their Pauli masks: Q' = S_dag * Q * S
  - Updates the final tableau: U_C' = U_C * S_P
  - Corrects global_weight with s_absorption_phase()

// From peephole.cc -- the fusion branch
if (total == 0) {           // T + T_dag = identity cancellation
    deleted[i] = deleted[j] = true;  ++cancellations_;
} else {                    // |total| == 2: fuse into S, absorb downstream
    bool s_is_dagger = (total == -2);
    deleted[i] = deleted[j] = true;
    apply_virtual_s_downstream(hir, j + 1, destab_i, stab_i,
                               false, s_is_dagger, deleted);
    ++fusions_;
}

Peephole: Rotation Folding

BEFORE:  Two arbitrary-angle rotations on same Pauli axis
+------------------+-------+------------------+
| PHASE_ROT(P, 0.3)| M(Q)  | PHASE_ROT(P, 0.2)|
|  alpha = 0.3 * pi| commu-|  alpha = 0.2 * pi|
|                  | tes   |                  |
+------------------+-------+------------------+

AFTER:   Fused into single rotation
+-------+------------------+
| M(Q)  | PHASE_ROT(P, 0.5)|
|       |  alpha = 0.5 * pi|
+-------+------------------+

Special cases detected automatically:
  a+b = 0.0  --> Identity (delete both)        ++cancellations_
  a+b = 0.5  --> S gate (absorb downstream)    ++fusions_
  a+b = 1.5  --> S_dag gate (absorb downstream) ++fusions_
  a+b = 0.25 --> T gate (demote to cheaper T)   ++fusions_
  a+b = 1.75 --> T_dag gate (demote)             ++fusions_
  otherwise  --> Keep as PHASE_ROT(a+b)          ++fusions_

Peephole: Fixed-Point Iteration & Sign Normalization

Fixed-Point Loop

bool changed = true;
while (changed) {
    changed = false;
    // 1) Scan for T gate pairs
    // 2) Scan for PHASE_ROT pairs
    // 3) Demote standalone
    //    PHASE_ROTs at Clifford/T angles
    // 4) Compact: erase deleted ops
}
// Repeats until no more
// cancellations/fusions found

A fusion may expose new cancellations -- e.g., after an S-absorption conjugates downstream T gates, a previously blocked pair may now match.

T Gate Sign Normalization

// T(-P) = exp(+i*pi/4) * T_dag(+P)
// T_dag(-P) = exp(-i*pi/4) * T(+P)
inline void normalize_t_sign(
    HirModule& hir, HeisenbergOp& op,
    std::complex<double>& global_weight)
{
    if (op.op_type() == OpType::T_GATE
        && hir.sign(op)) {
        global_weight *=
          op.is_dagger()
            ? kExpMinusIPiOver4
            : kExpIPiOver4;
        op.set_dagger(!op.is_dagger());
        hir.set_sign(op, false);
    }
}

All T gates are normalized to sign=false before comparison. The phase difference is absorbed into global_weight.

StatevectorSqueezePass

HIR DEFAULT ON

THE most important pass for performance: minimize peak active dimension k

Squeeze: Why Peak Rank Matters

The Central Insight

Every array operation costs O(2^k). Peak rank k determines the worst-case memory and runtime for the entire simulation. Reducing k by just 1 gives a 2x speedup. Reducing by 3 gives 8x.

What Changes k?

OperationEffect on k
T_GATE, PHASE_ROTk -> k+1 (expand)
MEASURE (active qubit)k -> k-1 (contract)
Clifford gatesNo change (O(1))

The Strategy

  1. Sweep 1 (leftward): Bubble MEASUREs as early as possible --> free active dimensions sooner
  2. Sweep 2 (rightward): Bubble T_GATEs and PHASE_ROTs as late as possible --> defer expansions

Squeeze: The Two-Sweep Algorithm

void StatevectorSqueezePass::run(HirModule& hir) {
    // Sweep 1: Eager Compaction (leftward bubble of MEASUREs)
    for (size_t i = 1; i < hir.ops.size(); ++i) {
        if (hir.ops[i].op_type() != OpType::MEASURE) continue;
        size_t curr = i;
        while (curr > 0 && can_swap(hir.ops[curr - 1], hir.ops[curr], hir)) {
            std::swap(hir.ops[curr - 1], hir.ops[curr]);
            --curr;                     // bubble left as far as possible
        }
    }

    // Sweep 2: Lazy Expansion (rightward bubble of non-Clifford gates)
    if (hir.ops.size() >= 2) {
        for (size_t i = hir.ops.size() - 2;; --i) {
            auto t = hir.ops[i].op_type();
            if (t == OpType::T_GATE || t == OpType::PHASE_ROTATION) {
                size_t curr = i;
                while (curr < hir.ops.size() - 1
                       && can_swap(hir.ops[curr], hir.ops[curr + 1], hir)) {
                    auto nt = hir.ops[curr + 1].op_type();
                    if (nt == OpType::T_GATE || nt == OpType::PHASE_ROTATION)
                        break;          // don't swap two expanding gates
                    std::swap(hir.ops[curr], hir.ops[curr + 1]);
                    ++curr;             // bubble right
                }
            }
            if (i == 0) break;
        }
    }
}

Squeeze: Before vs After -- k Trajectory

BEFORE optimization:  T1 T2 M1 T3 M2 M3 T4 M4

 k   4 |            *
     3 |      *---*   *
     2 |   *--      *   *
     1 | *               *
     0 |*                  *
       +-------------------------------> time
       T1 T2 M1 T3 M2 M3 T4 M4
                  peak k = 4  -->  memory = 2^4 = 16 amplitudes

AFTER optimization:   M1 M2 T1 T2 M3 T3 T4 M4
   (Measures moved left, T gates moved right)

 k   4 |
     3 |
     2 |            *---*      *
     1 |         *--      *--*
     0 |*---*---*               *
       +-------------------------------> time
       M1 M2 T1 T2 M3 T3 T4 M4
                  peak k = 2  -->  memory = 2^2 = 4 amplitudes
                  4x speedup, 4x less memory!

Squeeze: Commutation Safety via can_swap()

The can_swap() Function (commutation.cc)

Checks three independent conditions -- ALL must pass:

1. Classical Dataflow

// Block if one op writes
// a measurement index that
// the other op reads
auto left_write =
  get_written_meas_idx(left, hir);
if (left_write.has_value()
    && accesses_classical_index(
         right, *left_write, hir))
    return false;

2. EXP_VAL Barrier

// EXP_VAL is a positional probe:
// user expects the expectation
// value at an exact circuit point
if (lt == OpType::EXP_VAL
    || rt == OpType::EXP_VAL)
    return false;

3. Quantum Commutativity (Symplectic Inner Product)

// Standard case: if Pauli operators anti-commute, ops cannot be swapped
return !anti_commute(hir.destab_mask(left), hir.stab_mask(left),
                     hir.destab_mask(right), hir.stab_mask(right));
// Special handling for NOISE ops: check all channels via noise_sites_anti_commute()

RemoveNoisePass

HIR OFF by default

Strip all stochastic noise for clean reference-shot extraction

RemoveNoisePass: Clean & Simple

Purpose

Removes all NOISE and READOUT_NOISE operations from the HIR, producing a clean noiseless circuit.

Used internally by compute_reference_syndrome() to extract noiseless reference detector parities for syndrome normalization.

When to Use

  • Computing reference syndromes (noiseless parity)
  • Testing gate correctness without noise
  • Benchmarking pure gate performance

Implementation (entire pass)

void RemoveNoisePass::run(HirModule& hir) {
    // Remove all noise ops
    std::erase_if(hir.ops, [](const HeisenbergOp& op) {
        return op.op_type() == OpType::NOISE
            || op.op_type() == OpType::READOUT_NOISE;
    });

    // Clear side tables
    hir.noise_sites.clear();
    hir.readout_noise.clear();

    // Replace arena (fixed-capacity)
    hir.noise_channel_masks = PauliMaskArena{};
    hir.source_map.clear();
}
BEFORE: T M N M N N T RN M
AFTER:  T M   M      T    M

N  = NOISE op     (stochastic Pauli channel)
RN = READOUT_NOISE (classical bit-flip)

DropNonUnitaryPass

HIR OFF by default

Extract the pure unitary skeleton for probability computation

DropNonUnitaryPass: Unitary Skeleton

Purpose

Drops ALL non-evolution operations, leaving only T_GATE and PHASE_ROTATION -- the pure unitary part of the circuit.

Not semantics-preserving! Only valid for computing exact basis_probabilities() where no measurements or noise are needed.

What Gets Dropped

DroppedKept
MEASURET_GATE
CONDITIONAL_PAULIPHASE_ROTATION
NOISE, READOUT_NOISE
DETECTOR, OBSERVABLE
EXP_VAL

Implementation

void DropNonUnitaryPass::run(HirModule& hir) {
    std::erase_if(hir.ops, [](const HeisenbergOp& op) {
        switch (op.op_type()) {
            case OpType::T_GATE:
            case OpType::PHASE_ROTATION:
                return false;   // KEEP
            default:
                return true;    // DROP
        }
    });

    // Clear all side tables
    hir.noise_sites.clear();
    hir.readout_noise.clear();
    hir.detector_targets.clear();
    hir.observable_targets.clear();
    hir.noise_channel_masks = PauliMaskArena{};
    hir.num_measurements = 0;
    hir.num_hidden_measurements = 0;
    hir.num_detectors = 0;
    hir.num_observables = 0;
    hir.num_exp_vals = 0;
    hir.source_map.clear();
}

The static_assert on NUM_OP_TYPES == 9 ensures this pass is updated whenever a new OpType is added to the HIR.

NoiseBlockPass

BYTECODE DEFAULT ON

Coalesce contiguous OP_NOISE into OP_NOISE_BLOCK for O(1) skip-ahead

NoiseBlock: Reducing Dispatch Overhead

The Problem

QEC circuits emit hundreds of consecutive OP_NOISE instructions (one per physical qubit per error round). Each OP_NOISE has its own instruction fetch, decode, and dispatch overhead.

The SVM's gap-based noise sampler can skip silent sites in O(1), but still pays per-instruction dispatch cost.

The Solution

Replace runs of consecutive OP_NOISE(site_i), OP_NOISE(site_i+1), ... with a single OP_NOISE_BLOCK(start, count).

The VM processes the entire block in one dispatch cycle using exponential gap sampling over the cumulative hazard table.

BEFORE:  50 individual noise instructions
+----------+----------+----------+     +----------+
| OP_NOISE | OP_NOISE | OP_NOISE | ... | OP_NOISE |
| site=0   | site=1   | site=2   |     | site=49  |
+----------+----------+----------+     +----------+
  50 instruction fetches, 50 decodes, 50 dispatches

AFTER:   1 block instruction
+-----------------------+
| OP_NOISE_BLOCK        |
| start=0, count=50     |
+-----------------------+
  1 fetch, 1 decode, O(1) skip-ahead per silent site

NoiseBlock: The Algorithm

void NoiseBlockPass::run(CompiledModule& module) {
    auto& old_bc = module.bytecode;
    std::vector<Instruction> new_bc;
    new_bc.reserve(old_bc.size());

    size_t i = 0;
    while (i < old_bc.size()) {
        if (old_bc[i].opcode != Opcode::OP_NOISE) {
            new_bc.push_back(old_bc[i]);
            ++i;
            continue;
        }

        // Found OP_NOISE -- scan for consecutive site indices
        uint32_t start_site = old_bc[i].pauli.cp_mask_idx;
        size_t run_len = 1;
        while (i + run_len < old_bc.size()
               && old_bc[i + run_len].opcode == Opcode::OP_NOISE
               && old_bc[i + run_len].pauli.cp_mask_idx == start_site + run_len) {
            ++run_len;
        }

        if (run_len == 1) {
            new_bc.push_back(old_bc[i]);  // Keep singleton as-is
        } else {
            new_bc.push_back(make_noise_block(start_site, run_len));  // Coalesce!
        }
        i += run_len;
    }
    old_bc = std::move(new_bc);
}

Key Requirement: Consecutive Site Indices

Sites must be numbered start, start+1, start+2, ... for the gap sampler to use the cumulative hazard table as a contiguous range. Non-consecutive sites break the run.

MultiGatePass

BYTECODE DEFAULT ON

Fuse star-graph CNOT/CZ chains into bitmask-driven multi-gate instructions

MultiGate: Star-Graph Fusion

Origin: The Pauli Localizer

The backend's localize_pauli() reduces multi-qubit Pauli strings by emitting sequences of CNOT/CZ gates that all share a common pivot qubit. This creates star-graph patterns in the bytecode.

BEFORE:  Star-graph CNOTs sharing target axis 3
+-------------------+-------------------+-------------------+-------------------+
| OP_ARRAY_CNOT     | OP_ARRAY_CNOT     | OP_ARRAY_CNOT     | OP_ARRAY_CNOT     |
| ctrl=0, tgt=3     | ctrl=1, tgt=3     | ctrl=4, tgt=3     | ctrl=5, tgt=3     |
+-------------------+-------------------+-------------------+-------------------+
  4 separate O(2^k) array sweeps

AFTER:   Single multi-gate instruction
+-----------------------------------+
| OP_ARRAY_MULTI_CNOT               |
| target=3, mask=0b00110011         |
|          bits: 5 4 _ _ 1 0       |
+-----------------------------------+
  1 array sweep with popcount-based parity check

CNOT Pattern

Multiple ARRAY_CNOT(ctrl_i, target) sharing the same target axis --> ARRAY_MULTI_CNOT(target, ctrl_mask)

Bitmask: bit i set = axis i is a control

CZ Pattern

Multiple ARRAY_CZ(ctrl, tgt_i) sharing the same control axis --> ARRAY_MULTI_CZ(ctrl, target_mask)

Bitmask: bit i set = axis i is a target

MultiGate: Bitmask Encoding & Self-Cancellation

// CNOT fusion with XOR-based self-cancellation detection
while (i < old_bc.size() && old_bc[i].opcode == Opcode::OP_ARRAY_CNOT
       && old_bc[i].axis_2 == shared_target) {
    ctrl_mask ^= 1ULL << old_bc[i].axis_1;  // XOR: pairs cancel!
    ++run_len; ++i;
}

if (ctrl_mask == 0) {
    // All gates cancelled (even number of same-axis CNOTs = identity)
} else if (run_len >= 2 && std::popcount(ctrl_mask) >= 2) {
    new_bc.push_back(make_array_multi_cnot(shared_target, ctrl_mask));
} else {
    // Single surviving gate: emit as plain CNOT
    uint16_t ctrl_axis = static_cast<uint16_t>(std::countr_zero(ctrl_mask));
    new_bc.push_back(make_array_cnot(ctrl_axis, shared_target));
}

Elegant XOR Trick

Because CNOT is self-inverse, XOR-ing the bitmask automatically detects and cancels pairs. If two CNOTs with the same control axis appear, they cancel to identity. The final bitmask contains only the surviving controls.

VM Execution

The VM computes parity via popcount(index & mask) & 1 in the inner loop -- O(1) per amplitude entry. One sweep replaces N separate sweeps.

ExpandTPass & ExpandRotPass

BYTECODE DEFAULT ON

Fuse OP_EXPAND + OP_ARRAY_T/ROT into single-pass instructions

ExpandT: Eliminating a Redundant Array Pass

The Pattern

Every T-gate injection requires: (1) OP_EXPAND to activate a dormant axis (k -> k+1, doubles array), then (2) OP_ARRAY_T to apply the T-phase on that axis. That is two separate O(2^k) sweeps.

BEFORE:  Two instructions, two array sweeps
+-----------------------------+-----------------------------+
| OP_EXPAND(axis=5)           | OP_ARRAY_T(axis=5)          |
|                             |                             |
| for i in 0..2^k:            | for i in 0..2^(k+1):        |
|   v[i + 2^k] = v[i]        |   if bit(i, axis_5):        |
|   gamma /= sqrt(2)         |     v[i] *= exp(i*pi/4)     |
+-----------------------------+-----------------------------+
 Sweep 1: copy+expand           Sweep 2: phase multiply

AFTER:   One fused instruction, one array sweep
+----------------------------------------------+
| OP_EXPAND_T(axis=5)                          |
|                                              |
| for i in 0..2^k:                              |
|   v[i + 2^k] = v[i] * exp(i*pi/4)           |
|   gamma /= sqrt(2)                          |
+----------------------------------------------+
 Single sweep: copy and phase in one iteration
 Saves one full O(2^k) memory traversal

ExpandT & ExpandRot: Implementation

ExpandTPass

void ExpandTPass::run(CompiledModule& mod) {
    size_t i = 0;
    while (i < old_bc.size()) {
        if (i + 1 < old_bc.size()
            && old_bc[i].opcode == OP_EXPAND) {
            uint16_t axis = old_bc[i].axis_1;
            Opcode next = old_bc[i+1].opcode;

            if ((next == OP_ARRAY_T
                 || next == OP_ARRAY_T_DAG)
                && old_bc[i+1].axis_1 == axis) {
                // Fuse!
                Opcode fused = (next == OP_ARRAY_T)
                    ? OP_EXPAND_T : OP_EXPAND_T_DAG;
                Instruction f{};
                f.opcode = fused;
                f.axis_1 = axis;
                new_bc.push_back(f);
                i += 2; continue;
            }
        }
        new_bc.push_back(old_bc[i]); ++i;
    }
}

ExpandRotPass

void ExpandRotPass::run(CompiledModule& mod) {
    size_t i = 0;
    while (i < old_bc.size()) {
        if (i + 1 < old_bc.size()
            && old_bc[i].opcode == OP_EXPAND) {
            uint16_t axis = old_bc[i].axis_1;

            if (old_bc[i+1].opcode == OP_ARRAY_ROT
                && old_bc[i+1].axis_1 == axis) {
                // Fuse with rotation angle
                new_bc.push_back(
                    make_expand_rot(axis,
                        old_bc[i+1].math.weight_re,
                        old_bc[i+1].math.weight_im
                    ));
                i += 2; continue;
            }
        }
        new_bc.push_back(old_bc[i]); ++i;
    }
}

Impact

Every T gate in the circuit benefits. In a distance-5 surface code with 50 T gates, this eliminates 50 separate O(2^k) array copies. The fused kernel arr[i + half] = arr[i] * exp(+-i*pi/4) runs in a single pass.

SwapMeasPass

BYTECODE DEFAULT ON

Fuse OP_ARRAY_SWAP + OP_MEAS_ACTIVE_INTERFERE into a single dispatch

SwapMeas: Eliminating the Pre-Measurement Swap

Why the Backend Emits SWAP + MEAS

Active X-basis measurement requires the measurement axis to be at position k-1 (the highest active axis) for contiguous array compaction. The backend emits an OP_ARRAY_SWAP to route the axis there, followed by OP_MEAS_ACTIVE_INTERFERE.

BEFORE:  Two instructions, two O(2^k) sweeps
+-------------------------------+-------------------------------+
| OP_ARRAY_SWAP(axis=2, axis=4) | OP_MEAS_ACTIVE_INTERFERE      |
|                               | axis=4, classical_idx=7       |
| for i in 0..2^k:              | for i in 0..2^(k-1):          |
|   swap bit 2 and bit 4 pairs |   v[i] += v[i + 2^(k-1)]     |
|   in the index                |   (fold + halve array)        |
+-------------------------------+-------------------------------+
 Sweep 1: reindex all pairs       Sweep 2: fold+halve

AFTER:   One fused instruction
+----------------------------------------------------+
| OP_SWAP_MEAS_INTERFERE                             |
| swap_from=2, swap_to=4, classical_idx=7            |
|                                                    |
| for i in 0..2^(k-1):                               |
|   reindex(i, axis_2, axis_4) then fold+halve      |
+----------------------------------------------------+
 Single sweep: reindex and fold in one pass

SwapMeas: Implementation

void SwapMeasPass::run(CompiledModule& module) {
    auto& old_bc = module.bytecode;
    std::vector<Instruction> new_bc;
    new_bc.reserve(old_bc.size());

    size_t i = 0;
    while (i < old_bc.size()) {
        if (i + 1 < old_bc.size()
            && old_bc[i].opcode == Opcode::OP_ARRAY_SWAP
            && old_bc[i + 1].opcode == Opcode::OP_MEAS_ACTIVE_INTERFERE
            && old_bc[i + 1].axis_1 == old_bc[i].axis_2) {

            new_bc.push_back(make_swap_meas_interfere(
                old_bc[i].axis_1,                      // swap_from
                old_bc[i].axis_2,                      // swap_to (= meas axis)
                old_bc[i + 1].classical.classical_idx, // measurement record slot
                (old_bc[i + 1].flags & Instruction::FLAG_SIGN) != 0
            ));
            i += 2;
            continue;
        }
        new_bc.push_back(old_bc[i]);
        ++i;
    }
    old_bc = std::move(new_bc);
}

Pattern Match Condition

The SWAP's axis_2 (destination) must equal the MEAS's axis_1 (measurement target). This confirms the swap was routing the qubit to position k-1 for measurement.

TileAxisFusionPass

BYTECODE DEFAULT ON

Fuse 2-qubit tile sequences into precomputed 4x4 unitary matrices

TileAxisFusion: The Idea

Core Concept

When a sequence of gates all operate on the same pair of axes {a, b}, their combined effect is a single 4x4 unitary matrix. Instead of N separate O(2^k) array sweeps, apply one 4x4 matrix in a single sweep.

What Can Participate

1Q Array Ops2Q Array Ops
ARRAY_HARRAY_CNOT
ARRAY_S / S_DAGARRAY_CZ
ARRAY_T / T_DAGARRAY_SWAP
ARRAY_ROT

Frame-only ops (FRAME_H, FRAME_S, FRAME_CNOT, etc.) are also absorbed into the fused matrix.

Run Terminators

  • Instruction touching a different axis
  • OP_EXPAND (changes array dimension)
  • Measurement, noise, or classical op
  • Frame op that mixes tile axis with external axis

Minimum 3 array-touching ops to justify the 4x4 matrix overhead.

TileAxisFusion: Before/After

BEFORE:  5 instructions on axes {0, 1}  (5 separate O(2^k) sweeps)

| OP_ARRAY_CNOT    | OP_ARRAY_S       | OP_ARRAY_CZ      | OP_ARRAY_H       | OP_ARRAY_T       |
| ctrl=0, tgt=1    | axis=0           | a=0, b=1         | axis=1           | axis=0           |
+------------------+------------------+------------------+------------------+------------------+

| axis_lo=0, axis_hi=1, cp_idx=0              |
|                                              |
| ConstantPool.fused_u4_nodes[0]:             |

The FusedU4Node Structure

struct FusedU4Node {
    struct Entry {
        std::complex<double> matrix[4][4]; // Row-major 4x4 unitary
        std::complex<double> gamma_multiplier;  // Accumulated global phase
        uint8_t out_state;  // Resulting 4-bit frame: (pz_hi<<3)|(px_hi<<2)|(pz_lo<<1)|px_lo
    };
    Entry entries[16];  // Indexed by 4-bit incoming frame state
};
// 16 states = all combinations of (px_lo, pz_lo, px_hi, pz_hi)

TileAxisFusion: How U4 Nodes Are Built

FusedU4Node build_fused_u4_node(const std::vector<Instruction>& bytecode,
                                size_t start, size_t end,
                                uint16_t tile_lo, uint16_t tile_hi) {
    FusedU4Node node{};
    for (uint8_t in_state = 0; in_state < 16; ++in_state) {
        Mat4 M = kIdentity4;            // Start with 4x4 identity
        std::complex<double> gamma = {1.0, 0.0};
        uint8_t px_lo = in_state & 1;
        uint8_t pz_lo = (in_state >> 1) & 1;
        uint8_t px_hi = (in_state >> 2) & 1;
        uint8_t pz_hi = (in_state >> 3) & 1;

        for (size_t i = start; i < end; ++i) {
            simulate_tile_instruction(bytecode[i],
                px_lo, pz_lo, px_hi, pz_hi,
                M, gamma, tile_lo, tile_hi);
        }
        // Store the accumulated matrix, phase, and output frame state
        auto& entry = node.entries[in_state];
        for (int r = 0; r < 4; ++r)
            for (int c = 0; c < 4; ++c)
                entry.matrix[r][c] = M[r * 4 + c];
        entry.gamma_multiplier = gamma;
        entry.out_state = (pz_hi << 3) | (px_hi << 2) | (pz_lo << 1) | px_lo;
    }
    return node;
}

Each of 16 input frame states produces a different 4x4 matrix and phase. The VM reads (pz_hi, px_hi, pz_lo, px_lo) at runtime, looks up the precomputed entry, and applies a single 4x4 matrix multiply.

Ordering Constraint

TileAxisFusionPass MUST run BEFORE SingleAxisFusionPass. TileAxisFusion needs to see the raw primitive opcodes (ARRAY_CNOT, ARRAY_S, etc.). SingleAxisFusion then cleans up any leftover 1Q chains that were not part of a tile.

BYTECODE DEFAULT ON

Fuse single-axis gate chains into precomputed 2x2 unitary matrices

SingleAxisFusion: 1Q Chains to U2

Core Concept

When a sequence of gates all operate on the same single axis, their combined effect is a 2x2 unitary. The pass precomputes this matrix for all 4 possible Pauli frame states and stores it in the ConstantPool.

BEFORE:  4 instructions on axis 2  (4 separate O(2^k) sweeps)
+------------------+------------------+------------------+------------------+
| OP_ARRAY_H       | OP_ARRAY_T       | OP_ARRAY_S       | OP_ARRAY_H       |
| axis=2           | axis=2           | axis=2           | axis=2           |
+------------------+------------------+------------------+------------------+

AFTER:   1 fused U2 instruction  (1 sweep with 2x2 matrix)
+----------------------------------------------+
| OP_ARRAY_U2                                  |
| axis=2, cp_idx=0                             |
|                                              |
| ConstantPool.fused_u2_nodes[0]:             |

+----------------------------------------------+
  4 sweeps --> 1 sweep   (4x fewer memory passes)

The FusedU2Node Structure

struct FusedU2Node {
    std::complex<double> matrices[4][4]; // [frame_state][row-major 2x2]
    std::complex<double> gamma_multipliers[4]; // Global phase per state
    uint8_t out_states[4]; // Output frame: (pz << 1) | px
};
// 4 states = (px, pz): I=00, X=01, Z=10, Y=11

SingleAxisFusion: Building FusedU2Nodes

FusedU2Node build_fused_node(const std::vector<Instruction>& bytecode,
                             size_t start, size_t end) {
    FusedU2Node node{};
    for (uint8_t in_state = 0; in_state < 4; ++in_state) {
        Mat2 M = {1.0, 0.0, 0.0, 1.0};  // 2x2 identity
        std::complex<double> gamma = {1.0, 0.0};
        uint8_t px = in_state & 1;       // X bit of Pauli frame
        uint8_t pz = (in_state >> 1) & 1; // Z bit of Pauli frame

        for (size_t i = start; i < end; ++i) {
            simulate_instruction(bytecode[i], px, pz, M, gamma);
        }

        for (int j = 0; j < 4; ++j)
            node.matrices[in_state][j] = M[j];
        node.gamma_multipliers[in_state] = gamma;
        node.out_states[in_state] = (pz << 1) | px;
    }
    return node;
}

Frame-Aware Simulation

Each gate's action depends on the current Pauli frame. For example:

  • ARRAY_T with px=0: matrix diag(1, exp(i*pi/4))
  • ARRAY_T with px=1: matrix diag(1, exp(-i*pi/4)), plus gamma *= exp(i*pi/4)

This is why we need 4 separate precomputed matrices -- the effective unitary changes based on the runtime frame state.

SingleAxisFusion: Gate Simulation Details

void simulate_instruction(const Instruction& inst,
                          uint8_t& px, uint8_t& pz,
                          Mat2& M, std::complex<double>& gamma) {
    Mat2 U = {1.0, 0.0, 0.0, 1.0};  // identity

    switch (inst.opcode) {
        case OP_ARRAY_H:
            U = {kInvSqrt2, kInvSqrt2, kInvSqrt2, -kInvSqrt2};
            if (px && pz) gamma *= {-1.0, 0.0};  // Y -> -Y under H
            std::swap(px, pz);                    // H swaps X<->Z frame bits
            break;
        case OP_ARRAY_S:
            U = {1.0, 0.0, 0.0, kI};
            if (px) gamma *= kI;                  // X -> Y, pick up i phase
            pz ^= px;                             // S maps X -> Y (XOR Z with X)
            break;
        case OP_ARRAY_T:
            if (px) { U[3] = kExpMinusIPiOver4; gamma *= kExpIPiOver4; }
            else    { U[3] = kExpIPiOver4; }
            break;
        // ... FRAME_H, FRAME_S: update frame + gamma only (no matrix)
        case OP_FRAME_H:
            if (px && pz) gamma *= {-1.0, 0.0};
            std::swap(px, pz);
            return;  // no matrix multiply
    }
    mat_mul_left(M, U);
}

Architecture Key Points

  • Order matters: PeepholeFusion before Squeeze (reduces ops that Squeeze would re-arrange). TileAxisFusion before SingleAxisFusion (tile pass sees raw opcodes).

  • Source map preserved: Every pass maintains the bytecode-to-source-line mapping through merging and copying.
  • Fixed infrastructure: HirPass + BytecodePass base classes, PassManager chains, PassFactory for named construction.
